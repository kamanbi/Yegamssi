## 2. 🌍 글로벌 서비스 전략

### ■ 서비스 확장 단계

1차: 한국 (기상청 API 기반)  
2차: 미국 (NOAA / 무료 API)  
3차: 글로벌 확장 (무료 API 우선 적용)

👉 전략:
- **무료 API 기반으로 단계적 확장**
- 초기 안정성 확보 후 글로벌 확대

---

### ■ 날씨 API 전략

- 한국: 기상청 API
- 미국: NOAA / National Weather Service
- 글로벌: OpenWeather Free / 기타 무료 API

👉 기준:
- 무료 사용 가능
- 신뢰도 확보
- 지역별 커버리지

---

## 3. 🌐 다국어 및 로컬라이징 전략

### ■ 기본 정책

- 모든 UI 텍스트는 언어 선택에 따라 자동 번역
- 국가별 문화 고려 필수

---

### ■ 언어 지원 구조

- 기본: 한국어 / 영어
- 확장: 일본어 / 중국어 / 유럽권 언어

---

### ■ 번역 원칙

- 단순 직역 금지
- 국가별 표현 방식 반영

예:
- 한국 → 감성 중심
- 미국 → 직관적 / 기능 중심
- 일본 → 정중하고 부드러운 표현

---

### ■ 운세 로컬라이징 (핵심)

운세는 단순 번역이 아니라 **국가별 문화 기반 재해석 필요**

예:
- 한국 → 사주 / 띠 / 감성형
- 미국 → Horoscope (별자리 기반)
- 일본 → 오미쿠지 스타일
- 중국 → 명리 기반

👉 각 국가별 운세 로직 분리 설계
