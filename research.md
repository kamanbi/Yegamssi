# 리서치: 날씨 상세 페이지 검토 원복 기준

## 현재 상태
- 상세 날씨 페이지 1차 구현은 원복했다.
- 기존 날씨 화면은 기존처럼 현재 날씨, 상세 수치, 시간별 예보, 주간 예보를 한 화면에서 표시한다.
- 상세 페이지 라우트, 예특보 provider, 예특보 데이터 소스, 상세 화면 파일은 제거했다.
- 기상청 예특보 API와 공공데이터포털 바다낚시지수는 활용 승인된 상태지만 앱 코드에는 아직 반영하지 않는다.

## 현재 날씨 구조
- `currentWeatherProvider`가 현재 위치 날씨를 제공한다.
- `selectedLocationWeatherProvider`가 즐겨찾기/검색 선택 지역 날씨를 제공한다.
- `KmaDataSource`가 기상청 초단기실황, 초단기예보, 단기예보, 중기예보 일부를 통합한다.
- 대기질은 에어코리아 연동 데이터를 `WeatherEntity`에 병합한다.
- 캐시는 `WeatherCacheStore`를 통해 유지하며, 신규 데이터 일부 실패 시 기존 예보/대기질을 보존한다.

## 확인된 제약
- 위치별 특보, 전국 전망, 바다낚시지수, 물때는 현재 앱 화면에 연결되어 있지 않다.
- 상세 페이지를 다시 추진하려면 기존 날씨 provider와 분리된 상세 전용 provider가 필요하다.
- 해양/낚시 기능은 날씨 상세와 성격이 달라 별도 기능으로 설계하는 것이 안전하다.

## 리스크
- 기존 화면에 상세 기능을 직접 섞으면 화면 복잡도와 provider 책임이 커진다.
- 예특보/낚시지수/물때는 API 실패 가능성이 있으므로 섹션 단위 fallback과 캐시가 필요하다.
- 현재 저장소에 Git 메타데이터가 없어 변경 원복은 수동 확인이 필요하다.
